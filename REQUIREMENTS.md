# VocalTune 機能要件書（技術非依存版）

本書は VocalTune を **別の技術スタック／別プラットフォームで再実装** する際の仕様書である。実装技術（Next.js / Web Audio API / Socket.io / IndexedDB 等）に依存せず、プロダクトとして満たすべき機能と振る舞いを定義する。

---

## 0. プロダクト概要

### 0.1 目的
DTM 制作者・ボーカリストが、**リアルタイムに自分の歌声のピッチを可視化** し、お手本メロディと比較・採点しながら練習できるトレーナーアプリを提供する。

### 0.2 主要ユースケース
1. **ソロ練習**: 1 人で歌い、現在ピッチを見ながら音程感を鍛える。
2. **採点練習**: 与えられたメロディ（楽譜）をなぞって歌い、精度スコアを得る。
3. **二人同時練習**: 離れた 2 人が同じ画面を共有し、お互いのピッチを見ながら共同編集する。
4. **成長確認**: 過去の練習スコアを振り返り、上達を確認する。

### 0.3 ターゲットユーザー
- DTM（デスクトップミュージック）制作者
- ボーカルレッスン受講者・指導者
- カラオケ上達目的の一般ユーザー

### 0.4 プラットフォーム非依存方針
本要件は Web／デスクトップ／モバイルのいずれで実装されてもよい。ただし「マイク入力」「リアルタイム描画」「ネットワーク同期」「永続化」が可能な環境を前提とする。

## 1. 音声入力

### FR-IN-01  マイク入力取得
ユーザーのマイクからリアルタイムに音声波形を取得できること。

### FR-IN-02  生音優先
OS／プラットフォーム側のエコーキャンセル・ノイズサプレッション・自動ゲイン調整は **すべて無効化** すること。これはピッチ検出精度を毀損するため。無効化できない場合、その旨をユーザーに警告する。

### FR-IN-03  サンプリングレート
44.1 kHz または 48 kHz を想定。これより低いサンプリングレートでは要件 FR-PIT-02 を満たせない可能性がある。

### FR-IN-04  明示的な開始
マイク入力は **ユーザーの明示操作（STARTボタン等）でのみ** 開始すること。ページ読込み時の暗黙起動は禁止。

### FR-IN-05  権限拒否ハンドリング
マイク権限が拒否された場合、理由をユーザーに提示し再許可手順を案内すること。

---

## 2. ピッチ検出

### FR-PIT-01  リアルタイム基本周波数推定
マイク入力波形から現在の基本周波数（F0）を継続的に推定すること。

### FR-PIT-02  検出レンジ
**60Hz〜1200Hz**（およそ C2〜C6）の人声音域をカバーすること。

### FR-PIT-03  検出精度
安定発声時のセント誤差が **±5 セント以内** に収まるアルゴリズムを採用すること。実装は自由（YIN / pYIN / CREPE / SWIPE 等）。

### FR-PIT-04  検出レイテンシ
マイクから画面表示までのレイテンシは **100ms 以下** を目標とする。

### FR-PIT-05  無音判定
信号レベル（RMS）が閾値未満のフレームは「無音」として扱い、ピッチ値を出力しないこと。閾値は環境ノイズに応じて動的に更新可能であること。

### FR-PIT-06  ジッター除去
短時間のピッチ揺らぎ（1 フレームの誤検出）を抑制するスムージング機構を持つこと。ただし、ビブラートなど意図的な揺らぎは潰さないこと。

### FR-PIT-07  音名・セント差出力
推定 F0 から **音名（C4 等）** および **最寄り音からのセント差（−50〜+50）** を導出できること。

---

## 3. ビブラート検出

### FR-VIB-01  検出条件
以下をすべて満たす区間をビブラートとして認識すること。
- 周期レート: 4〜8 Hz
- 振幅: ±20〜100 セント
- 最低 3 サイクル連続

### FR-VIB-02  採点への影響
ビブラート区間は **ピッチ中心値** で採点し、振幅そのものをペナルティにしないこと。

---

## 4. 環境ノイズキャリブレーション

### FR-CAL-01  ノイズ計測
任意のタイミングで「無音時のノイズレベル」を数秒間計測できること。

### FR-CAL-02  閾値自動調整
計測結果から FR-PIT-05 の無音判定閾値を自動算出・適用すること。

### FR-CAL-03  UI
計測中は進捗を表示し、完了後にノイズレベルと推奨閾値をユーザーに提示すること。

---

## 5. メトロノーム

### FR-MET-01  クリック音生成
BPM に同期したクリック音を生成できること。

### FR-MET-02  アクセント
各小節頭は他の拍と **区別可能な音（周波数・音色）** で鳴動すること。

### FR-MET-03  タイミング精度
音源生成はサンプル単位でスケジューリングされ、時間ドリフトを持たないこと（汎用タイマーによる駆動は不可）。

### FR-MET-04  UI 制御
ON/OFF のトグルと BPM 変更が即座に反映されること。

---

## 6. 楽譜（お手本メロディ）

### FR-SCR-01  楽譜ファイル読込み
標準フォーマット（最低限 **MIDI 形式**）の楽譜ファイルを読み込めること。将来的に MusicXML や独自 JSON への拡張を許容する設計とすること。

### FR-SCR-02  読込み導線
ファイル選択ダイアログおよびドラッグ＆ドロップの双方に対応すること。

### FR-SCR-03  オーバーレイ表示
読み込んだ楽譜のノートをピアノロール上に **半透明矩形** として表示すること。

### FR-SCR-04  エラーハンドリング
パース失敗時は既存の練習状態を破棄せず、人間が読めるエラーを提示すること。

### FR-SCR-05  デモ楽譜同梱
ユーザーが外部ファイルを持たなくても体験できるよう、**C メジャースケール（C4〜C5）等のデモ楽譜** を同梱し、オンボーディング完了後に自動ロードすること。

---

## 7. ピアノロール描画

### FR-ROL-01  座標系
- 横軸: 時間（拍ベース、BPM・小節数に連動）
- 縦軸: 半音（固定高さ）

### FR-ROL-02  リアルタイム描画
現在ピッチを **60fps 目標** で滑らかに描画すること。

### FR-ROL-03  ピッチ履歴
直近の一定時間分のピッチ軌跡を折れ線として表示し、時間経過で流れていくこと。固定長バッファ（概ね 200 フレーム相当）で古いサンプルを破棄する。

### FR-ROL-04  補間
ピッチ表示は線形補間等で視覚的に滑らかにすること（カクつき禁止）。

### FR-ROL-05  音名カラーリング
各音名ごとに背景色を色分けすること。参考配色:
C=赤 / D=黄 / E=緑 / F=橙 / G=水 / A=紫 / B=白

### FR-ROL-06  グリッド
拍・小節の垂直グリッド、半音の水平グリッドを描画すること。

### FR-ROL-07  解像度追従
ウィンドウリサイズおよび高 DPI ディスプレイに追従して再描画されること。

### FR-ROL-08  縦解像度調整
半音あたりのピクセル高さをユーザーが変更できること。

---

## 8. 目標ピッチ編集モード

### FR-EDT-01  モード切替
編集モードの ON/OFF をトグルできること。

### FR-EDT-02  描画入力
マウス／タッチのクリック・ドラッグによって目標ピッチラインを自由に描画できること。

### FR-EDT-03  縦スナップ
Y 軸は必ず半音単位にスナップされること。

### FR-EDT-04  横スナップ
X 軸（拍）のグリッドスナップを ON/OFF トグルで切替できること。

### FR-EDT-05  複数ストローク
描画は独立した「ストローク」単位で保持され、同一座標を横切る新規ストロークは既存点を置換／消去すること。

### FR-EDT-06  Undo/Redo
ストローク単位で Undo/Redo できること。ショートカット推奨（Ctrl+Z / Ctrl+Shift+Z）。履歴は最低 50 件保持。

### FR-EDT-07  全消去
全ストロークを一括削除する操作を提供すること。

---

## 9. 採点

### FR-EVL-01  区間ごとの評価
お手本の各ノート区間ごとに、検出ピッチとのセント差から精度スコアを算出すること。

### FR-EVL-02  集計
区間スコアを **加重平均**（区間長に比例）してセッションスコア（0〜100）を算出すること。

### FR-EVL-03  ビブラート考慮
FR-VIB-02 に従い、ビブラート区間は中心周波数で評価する。

### FR-EVL-04  結果表示
セッション終了時にスコアと区間別内訳を表示し、保存可否をユーザーに委ねること。

### FR-EVL-05  セント差の可視化
現在ピッチと最寄り音とのセント差を **メーター UI** でリアルタイム表示すること（−50〜+50 の目盛）。

---

## 10. ソロモード

### FR-SOL-01  開始／停止
START / STOP ボタンで練習セッションを開始・終了できること。

### FR-SOL-02  現在情報表示
音名・周波数（Hz）・セント差を常時表示すること。

### FR-SOL-03  保存
セッション終了時、タイトル・タグを付与して練習記録を保存できること。

### FR-SOL-04  リセット
履歴を破棄して次のセッションを開始できること。

---

## 11. 二人同時練習（Co-op）モード

### 11.1 ルーム管理
- **FR-COP-01**: 「ルーム作成」で一意な ID を生成し、作成者が自動参加すること。
- **FR-COP-02**: 他ユーザーは **招待 URL** または **Room ID 手入力** で参加できること。
- **FR-COP-03**: 1 ルーム最大 **2 名**。3 人目の参加要求は拒否されること。
- **FR-COP-04**: 切断・タイムアウトを検出し、相手に通知すること。
- **FR-COP-05**: 全員退出から一定時間（推奨 5 分）経過後にルームを自動破棄すること。
- **FR-COP-06**: 招待 URL をクリップボードへコピーする機能を提供すること。

### 11.2 同期対象
以下を 2 ユーザー間でリアルタイム同期すること。

| 種別 | 項目 | 同期方式 |
|---|---|---|
| スカラー | BPM / 小節数 / 半音高さ / メトロノーム ON/OFF / 編集モード ON/OFF | Last-writer-wins 即時 |
| ストローク | 目標ピッチ描画（ユーザー別色分け） | スロットル（〜30ms）送信 |
| ファイル | 楽譜（MIDI 等）のバイナリ | 読込み時に一括転送、受信側で再パース |
| インタラクション | カーソル座標 | 高頻度送信（軽量） |
| 録音状態 | 開始／停止／ピッチサンプル | イベント送信 |

### 11.3 視覚表現
- **FR-COP-07**: 自分と相手のストロークは **異なる色** で描画すること（推奨: 自分=黄、相手=シアン）。
- **FR-COP-08**: 相手のカーソル位置を十字＋ドット等で描画すること。

### 11.4 接続状態 UI
- **FR-COP-09**: 接続中／切断／再接続中のステータスを明示すること。
- **FR-COP-10**: 再接続は自動試行すること。

### 11.5 競合解決
- **FR-COP-11**: スカラー値は Last-writer-wins で十分。CRDT 等の高度な整合性モデルは **非要件**。
- **FR-COP-12**: ストロークは追加型（append-only）を基本とし、消去は明示的操作としてイベント化する。

## 12. 3D ビジュアライザー

### FR-3D-01  提供
ピッチをリアルタイムに 3D 可視化する別画面を提供すること。

### FR-3D-02  表現要素
- 現在ピッチを表すオーブ／点
- 直近履歴のリボン（時間軸で奥行方向へ流れる）
- 半音グリッドの 3D 表示

### FR-3D-03  代替手段
プラットフォーム上で 3D が困難な場合、同等の視覚的ダイナミズムを持つ代替（派手な 2D アニメーション等）で代用してよい。

---

## 13. 練習履歴

### FR-HIS-01  永続化
練習記録を **ローカル永続ストレージ** に保存できること。クラウド同期は非要件。

### FR-HIS-02  保存スキーマ
少なくとも以下を保持すること。
- タイトル（ユーザー入力）
- スコア
- ピッチ履歴（時系列サンプル）
- 楽譜ファイル名または識別子
- タグ（複数可）
- 作成日時

### FR-HIS-03  一覧表示
保存済み記録を新しい順に一覧表示し、削除操作を提供すること。

### FR-HIS-04  再生／再描画
過去のピッチ履歴をピアノロール上に再描画できること（音声再生は任意）。

### FR-HIS-05  スコア推移グラフ
スコアを時系列の折れ線グラフで表示すること。

### FR-HIS-06  フィルタ
- 期間フィルタ: 7 日 / 30 日 / 全期間
- 楽譜フィルタ: 同一曲のみ抽出

---

## 14. ホーム画面

### FR-HOM-01  モード選択
Solo / Co-op の明確な導線を提示すること。

### FR-HOM-02  Co-op 導線
「ルーム作成」「Room ID で参加」の 2 系統を提供すること。

### FR-HOM-03  キャリブレーション導線
マイクキャリブレーションをホームから起動できること。

### FR-HOM-04  履歴導線
練習履歴画面への導線を提供すること。

## 15. オンボーディング

### FR-ONB-01  初回ガイド
初回起動時に 3 ステップ程度のガイドを自動表示すること。
1. マイク許可の取得
2. 基本的な使い方（START／ピアノロールの読み方）
3. 採点の読み方（お手本の重ね方／スコアの意味）

### FR-ONB-02  再表示
設定画面等から任意に再表示できること。

### FR-ONB-03  デモ楽譜自動ロード
ガイド完了時にデモ楽譜（FR-SCR-05）を自動ロードし、ユーザーが即座に体験できること。

### FR-ONB-04  一度きりのフラグ管理
「初回かどうか」の判定はローカル永続ストレージで行うこと。

---

## 16. オフライン対応

### FR-OFF-01  オフライン起動
オフライン状態でもアプリ本体が起動し、ソロモードの主要機能（マイク入力・ピッチ検出・ピアノロール・採点・履歴保存）が動作すること。

### FR-OFF-02  Co-op のオフライン
Co-op モードはネットワーク必須。オフライン時はその旨を明示すること。

## 17. 非機能要件

### NFR-01  パフォーマンス
- 1 フレーム処理時間の目標: **16ms 以下**（60fps）
- ピッチ検出→表示レイテンシ: **100ms 以下**
- メモリ: 長時間セッション（60 分）でリークしないこと

### NFR-02  UI
- ダークテーマを基調とすること
- コードフォントを活用した DTM 向けの見た目

### NFR-03  アクセシビリティ
- 主要操作はキーボードから可能なこと
- カラーのみに依存しない情報伝達（音名はテキストでも表示）

### NFR-04  国際化
- 文言は最低限日英切替可能な構造にすること（MVP では日本語のみでも可）

### NFR-05  デバイス
- デスクトップおよびタブレットでの横画面利用を主要ターゲットとする
- スマートフォン縦画面は副次的対応でよい

### NFR-06  セキュリティ／プライバシー
- 音声データは **ローカル完結**。外部送信は Co-op の同期データに限定すること
- 録音データがローカルに保存される旨をユーザーに明示すること

### NFR-07  拡張性
- 楽譜フォーマット（MIDI／MusicXML／独自 JSON）を差し替え可能な構造とすること
- ピッチ検出アルゴリズムを差し替え可能な構造とすること

---

## 18. スコープ外（非要件）

以下は本 MVP の範囲外。将来検討項目として記す。

- 3 人以上の多人数 Co-op
- 和音・ハモリ採点
- クラウドアカウント・クラウド同期
- 歌詞同期表示
- リバーブ等のエフェクト追加
- 音声そのものの録音・書き出し（現状はピッチ履歴のみ保存）
- モバイルネイティブ特化機能（バックグラウンド録音等）

---

## 付録 A: 最低限達成すべき機能優先度

再実装時、以下の優先順位で段階的に構築することを推奨する。

1. **P0（必須）**: FR-IN-*, FR-PIT-*, FR-ROL-*, FR-SOL-*, FR-ONB-*
2. **P1（重要）**: FR-SCR-*, FR-EVL-*, FR-HIS-*, FR-MET-*, FR-CAL-*, FR-EDT-*
3. **P2（差別化）**: FR-COP-*, FR-VIB-*, FR-3D-*
4. **P3（品質）**: NFR-*, FR-OFF-*

以上。
